# netsim

`netsim` is a concrete project skeleton for a web-controlled network simulator on Ubuntu 22.04/24.04.
It models two services:

- `netsim-api`: unprivileged FastAPI control plane
- `netsim-agent`: privileged agent that reconciles Linux networking state

The skeleton focuses on three things:

- typed API schemas for lines, profiles, interfaces, and command plans
- a service layout that cleanly separates control plane from privileged execution
- an explicit `tc` and `nft` command planner for both routing and bridge mode

## Layout

```text
netsim/
  README.md
  pyproject.toml
  docs/
    api-schema.md
    command-model.md
  src/
    netsim_common/
      models.py
      profiles.py
    netsim_api/
      main.py
      state.py
      routers/
        interfaces.py
        lines.py
        profiles.py
    netsim_agent/
      planner.py
      service.py
      profiles/
        defaults.json
```

## Run The API Skeleton

```bash
uvicorn netsim_api.main:app --app-dir netsim/src --reload
```

## Intended Runtime Split

- `netsim-api` owns validation, REST, profile selection, and desired-state persistence.
- `netsim-agent` owns `tc`, `ip`, `bridge`, `nft`, `ethtool`, and `sysctl`.
- The API should talk to the agent over a local Unix socket or loopback-only RPC endpoint.

## Notes

- This skeleton does not apply host networking changes by itself.
- The command planner emits the exact command sequences the future agent should reconcile.
- Interface names generated by the planner are hashed to stay within Linux name-length limits.
